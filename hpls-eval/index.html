HPLS-EVAL

Hardware-aware evaluation notes

HPLS-Eval frames precision, latency, hardware, and stability as review conditions for segmentation output behavior. It keeps the evaluation plan separate from completed model claims.

01

Precision

Track FP32, FP16, and INT8 conditions with runtime metadata and fallback status.

02

Latency

Record timing context carefully, including backend and device lane, before comparing rows.

03

Stability

Compare mask drift, boundary behavior, components, and centroid movement rather than relying on a single score.

How to use this workspace

  • Start from a documented FP32 reference path.
  • Add FP16 or INT8 rows only when the runtime path is clear.
  • Keep masks, logs, thresholds, and backend notes together.
  • Report untested hardware as planned, not measured.
